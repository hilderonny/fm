Login

First of all the user has to make a connection to Webdav server using Windows File Explorer by specifying the webdav server Url wich is \\localhost@SSL@56789\DavWWWRoot in the windows explorer. After the connection is done successfully, the user cradintial must be entered to browse and retrieve files and documents. The procedures is explanied as below in details.

    1. From the Start Menu on Windows PC go to File Explore and right click on This PC.
    2. Select map network drive from the drop list
    3. From the pop up window, Enter \\localhost@SSL@56789\DavWWWRoot in the folder filed.
    4. click Finish

The login window will pop up after clicking on th finish button in the previous step.The user and password has to be entered in order to view all the folder and documents belong to the loged in user credentials. Keep in mind that sometimes the login window takes a couple of times to get opened.

Attention!
Single connection is supported which means multiple connection to a server using different user credentials are not allowed. However, it is possible to maka a connection with other credentials by disconnecting the previous one.




To logout means to disconnect to the WevDav server. The following steps explain how to logout:

    1. From the Tree View in Windows Explorer go to the WebDav url name and right click on it.
    2. From the drop list select Disconnect.
    3. Click Ok to confirm the disconnection otherwise click on Cancel.
    4. Click Finish
    5. Refresh the browser to see the changes

Change credentials mean that the user can login with different credentials. However, after disconneting the previous connection to webdav. The same login procedures as explained above must be followed but with one more step which is checking on connect using different credentials.

Specifics

Some specifics are provided in this system such as nested/ring hierarchy,relevant functionality,update on-demand.

    1. Nested Hirarchy: a same folder or document with a same name can be appeared several times at the same or different folder list in the browser depends on its dependency. it is crucial, to know that this document or folder is not a copy of the original one but the it is the orignal itself. Therefore, any updates made in one of them will be made to othere one of the same folder or document
    2. Update on-Demand:when any changes are made in the web application after login to webdav, will not get the last updates in webdav unless the browser is refreshed. For example if an image is deleted in the web application and the user forgot to refresh the browser, as a sequence the image in webdav won't be deleted and displayed as well. Refresh the browser regularly!

Limitations

Deleting,creating,and renaming a folder/document are not supported. Regarding the deletion, the user can delete a folder or a file but this is not a real deletion because the deleted file or folder will be shown again in the borwser since it has been refreshed or the user logged in again. This occured because the deletion does not effect the database. Error messges is appered when trying to creat or rename a folder or a document.This messages has to be ignored.
