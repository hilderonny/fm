Hilfe zu WEbDAV

WebDav can be used for file browsing and document retrieval remotly over the internet using Windows file explorer. This module provides an alternative visulaization of the content of the documents module, but suppurts only a subset of possible interactions. Documents and folders can be viewed, renamed and retrieved, but they cannot be deleted or reallocated. The creation of new documetns and folders is currently also not supported.

Login

Simmilar to the user login procedure for the web application, the data access via WebDav server happens after user authentication. First of all, the user has to make a connection to WebDav server using the Windows map network drive interface. This is done by specifying the webdav server Url, which has the following format: \\hostName@SSL@portNumber\DavWWWRoot. In the particular case, the hostName is TODO and the portNumber is 56789 . After the connection is done successfully, the user cradintial must be entered. The full login procedure is explanied in details below.

    1. From the Start Menu on Windows PC go to File Explore and right click on This PC.
    2. Select map network drive from the drop list
    3. From the pop up window, Enter \\hostName@SSL@portNumber\DavWWWRoot in the folder filed.
    4. click Finish

The login window will pop up after clicking on th finish button in the previous step.The user name and password has to be entered in order to complete the user authentication. Keep in mind that sometimes the login window takes a couple of seconds to get opened.

Attention!
Due to security reasons, the Windows WebDav client only supports single-user connection. This means that multiple connections to a server using different user credentials are not allowed. However, it is possible to make a connection with different credentials by disconnecting the previous one and reconnecting with the new credentials.



To logout means to disconnect from the WevDav server. The following steps explain how to logout:

    1. From the list of Network locations in Windows file explorer right click on the currently mapped WebDav shared drive.
    2. From the drop-down list select Disconnect.
    3. Click Ok to confirm the disconnection.
    4. Click Finish.
    5. Refresh the browser to see the changes

Changing credentials means that the user can login as a different user. However, this can happen only after disconneting from the current WebDav connection. Afterwards, the same login procedures as explained above must be followed. The one significant difference here is that users should check the connect using different credentials checkbox (as shown on the image).

Specifics

Users should be aware of the following system behavior:

    1. Nested Hirarchy: the same folder or document with the same name can appear repeatedly at different locations in the hierarchy. This design is different than the typical Windows file system, which most users are familiar with.
    2. Update on-Demand:when any changes are made in the web application after login to WebDav, these will not automatically appear in the WebDav folder. The updated become visible only after the user manually refreshes the file browser. For example if an image is deleted in the web application but the user forgot to refresh the file browser, an attempt to open this deleted image will result in a error message or a low quality cashed preview. Please, refresh the browser regularly!

Limitations

The deletion, creation, and reallocation of a folder/document is not supported. Regarding the deletion, due to the automatic bahviour of the Windows WebDav cleint, the user may try to delete a folder or a document and get misleading visual feedback. It will look as if the deletion has been completed, although in reallity this action is prohibited by the server. Afer refreshing the file browser, the temporarily "deleted" data item will reapear again. Additionally, error messges will apper when the user tries to creat or reallocate a folder or a document. Such messages should be ignored.
